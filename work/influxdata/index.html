INTERNSHIP AT INFLUXDATA

from

June 2020 - August 2020

context

FLUX is InfluxData’s new functional data scripting language designed for querying, analyzing, and acting on data.

It is mainly used to interact with InfluxDB which is the real-time database by InfluxData.

Learn more here: https://github.com/influxdata/flux

1st Contribution

Redesigning the Flux language TypeExpression syntax.

To make the flux language syntax more user-friendly, I worked with the full time engineers at InfluxData flux team and devised a cleaner, better syntax which was more intuitive and user-centered.

before
"hour" =>
"forall [t0] where t0 : Timeable (t: t0) -> int"
after
"hour" =>
"(t: T) => int where T: Timeable"

2nd Contribution

Implementing a new parser for the new grammar.

I implemented a full recursive descent parser from ground up to be able to use the new syntax. This included implementation of the parser and the tests for individual grammar rules.

There were also times where I had to revisit the grammar to make it Left Factored for a smooth implementation.

3rd Contribution

Improving the developer’s contrib flow by optimizing compiler.

One of the main selling points / attraction for the Flux language is it’s open-source nature. This means that the codebase can be contributed-to by other developers.

I improved the contrib by optimizing the compiler. Previously developers would have to append their “modules” to two different files. This caused confusion because they would have to deal with understanding new syntaxes and standards.

After my optimization, the developers were able to include their source implementations in one flux module which could then be appended to the official flux package if it was approved. That would be done by a simple drag and drop into the sources folder.
